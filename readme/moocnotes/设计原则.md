## 开闭原则
1. 定义：一个软件实体如类、模块和函数应该对扩展开放，对修改关闭。
2. 用抽象构建框架，用实现扩展细节。
3. 优点：提高软件系统的可复用性及可维护性。
4. 案例：比如互联网公司的弹性工作制，每天工作八小时是定的，什么时候上班，什么时候下班是可以变动扩展的。
5. 实现开闭原则的核心思想就是面向抽象编程，如何从业务场景中抽象出来业务模型，并且从抽象化得出具体化的实例。
